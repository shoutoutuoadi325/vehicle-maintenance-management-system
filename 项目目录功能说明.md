# 车辆维修管理系统目录功能说明

## 1. 项目整体定位
本仓库是一个前后端分离的车辆维修管理系统，包含：
- 业务数据库设计与初始化脚本
- Java Spring Boot 后端（REST API + 业务规则 + 持久化）
- Vue 2 前端（多角色界面）
- 端到端测试与运行说明文档

系统在原始维修业务基础上，扩展了绿色导向（碳排放评估、减排奖励）、游戏化（里程/问答/奖励）以及 AI 故障诊断能力。

## 3. backend/（后端服务）
backend 是系统核心业务层，负责认证授权、工单流程、自动分配、绿色规则、游戏化、AI 诊断、数据持久化等能力。

### 3.1 工程与构建目录
- `pom.xml`
  - Maven 构建描述文件。
  - 声明主要技术栈：Spring Boot Web、JPA、Validation、Cache、MySQL、JWT、OpenAPI、OkHttp 等。
- `mvnw` / `mvnw.cmd`
  - Maven Wrapper 脚本，保证不同机器使用一致 Maven 版本。
- `.mvn/wrapper/`
  - Wrapper 配置目录，包含下载 Maven 发行版所需配置。
- `HELP.md`
  - Spring Boot 模板生成的基础帮助文档。

### 3.2 专题设计文档
- `AI_DIAGNOSIS_README.md`
  - AI 故障诊断接口说明、环境变量配置与测试方式。
- `EVENT_DRIVEN_DESIGN.md`
  - 事件驱动解耦设计说明，描述“工单完成 -> 减排事件 -> 游戏化奖励”的单向注入模型。
- `绿色导向功能说明.md`
  - 绿色维修决策辅助功能说明，解释碳排放估算字段、计算逻辑、测试建议。

### 3.3 源码目录：src/main/java/org/com/repair/
这是后端业务代码主目录，采用典型分层架构。

- `BackendApplication.java`
  - Spring Boot 启动入口。

- `controller/`
  - 控制器层，负责 HTTP API 入参接收、基础校验、调用 service、返回响应。
  - 覆盖模块：用户、管理员、技师、工单、反馈、材料、认证、游戏化、AI 诊断、全局异常处理。

- `service/`
  - 业务服务层，承载核心规则与事务逻辑。
  - 典型职责：
    - 工单创建/流转/完成结算
    - 技师自动分配
    - 绿色排放计算
    - 游戏化奖励发放
    - 鉴权令牌处理
    - AI 诊断请求封装与外部调用

- `repository/`
  - 数据访问层（Spring Data JPA Repository）。
  - 提供实体的查询与持久化接口，隔离 SQL 细节。

- `entity/`
  - 持久化实体模型，对应数据库表结构。
  - 包含传统维修业务实体（用户、车辆、工单等）与扩展实体（绿色账户、优惠券、旅程节点、刷新令牌等）。

- `DTO/`
  - 接口数据传输对象，隔离 API 结构与内部实体。
  - 作用：控制字段暴露范围、稳定前后端契约、支持请求参数校验。

- `config/`
  - 全局配置（如 CORS、缓存、拦截器注册）。

- `security/`
  - 安全相关组件。
  - 包含 JWT 解析、用户上下文解析、角色与资源归属权限拦截、安全审计等。

- `event/`
  - 领域事件对象定义（如减排事件），用于跨模块解耦通信。

- `exception/`
  - 业务异常与错误码定义，便于统一异常响应与错误语义。

### 3.4 资源目录：src/main/resources/
- `application.properties`
  - 后端运行配置中心。
  - 包含服务端口、MySQL 数据源、JPA 参数、AI 接口参数、JWT 参数等。

- `db/migration/`
  - 数据库增量迁移脚本目录（Flyway 约定命名）。
  - 反映项目版本演进：反馈评分字段、游戏化表、刷新令牌扩展、城市/随机事件、优惠券兑换字段、答错惩罚等。

### 3.5 测试目录：src/test/java/org/com/repair/
- 单元测试与契约测试主目录。
- 重点包含：
  - AI 诊断 API 可用性测试
  - 游戏化并发测试
  - 绿色排放计算测试
  - 全局启动测试与异常契约测试

### 3.6 构建产物目录：target/
- Maven 编译输出目录，属于可再生文件。
- 常见子目录：
  - `classes/`：编译后的主代码和资源
  - `test-classes/`：测试编译产物
  - `generated-sources/`：编译期生成代码
  - `maven-status/`：编译状态与增量信息
- 该目录通常不建议纳入版本控制。

## 4. frontend/（前端应用）
frontend 是基于 Vue 2 的单页应用，负责多角色交互界面与 API 调用。

### 4.1 工程与构建文件
- `package.json`
  - NPM 依赖与脚本入口。
  - 脚本：`serve`（开发运行）、`build`（生产构建）、`test:e2e`（端到端测试）。
- `package-lock.json`
  - 依赖锁定文件，保证安装一致性。
- `babel.config.js`
  - Babel 转译配置。
- `vue.config.js`
  - Vue CLI 项目配置（如 dev server、代理等）。
- `jsconfig.json`
  - JS 工程提示/路径配置，增强编辑器体验。
- `playwright.config.js`
  - E2E 测试框架 Playwright 的运行配置。
- `README.md`
  - 前端独立运行说明。

### 4.2 静态资源目录
- `public/`
  - 构建时原样拷贝的静态资源。
  - 包括入口模板 `index.html` 与网站图标 `favicon.ico`。

### 4.3 源码目录：src/
- `main.js`
  - 前端应用启动入口，初始化 Vue、路由、状态管理。
- `App.vue`
  - 根组件。

- `router/`
  - 前端路由定义，管理页面导航与访问入口。

- `store/`
  - Vuex 状态管理，集中维护跨页面共享状态。

- `components/`
  - 可复用组件层。
  - 典型组件：认证表单、图表组件、身份选择组件。

- `views/`
  - 页面级组件，按角色与业务场景组织。
  - 包含管理员、用户、技师看板，以及绿色旅程/大奖状态等页面。

- `assets/`
  - 图片、图标等前端资源。

### 4.4 测试目录：tests/e2e/
- 端到端测试脚本目录。
- 当前包含零碳旅程相关场景测试（如防抖行为验证）。

## 5. SQL/（数据库补充脚本）
该目录与后端迁移脚本形成互补，偏“手工初始化/数据库侧逻辑”。

- `Init/`
  - 初始数据脚本（如材料数据），用于快速构造基础业务数据。

- `Trigger/`
  - 数据库触发器脚本。
  - 例如删除用户、删除维修单时的级联清理逻辑，确保数据一致性。

## 6. 目录之间的协作关系

### 6.1 运行链路
1. 前端（frontend）发起 HTTP 请求。
2. 后端控制器（backend/controller）接收请求并分发到 service。
3. service 调用 repository 读写数据库。
4. 数据库结构由基础 SQL（根目录 + SQL）与增量迁移（backend/resources/db/migration）共同定义。
5. 业务事件（backend/event）把工单流程与游戏化奖励解耦。

### 6.2 数据模型治理链路
- 结构主线：`car_repair.sql` + Flyway 迁移脚本。
- 设计说明：`数据库表结构说明.md` 与 `E-R.svg`。
- 补充维护：`SQL/Trigger` 内触发器脚本。

### 6.3 测试链路
- 后端：`backend/src/test` 负责业务逻辑、并发与接口契约。
- 前端：`frontend/tests/e2e` 验证真实用户路径行为。

## 7. 维护建议（基于当前目录结构）

- 建议把敏感配置（数据库密码、AI Key、JWT Secret）完全迁移到环境变量，避免默认值落在代码仓库。
- 建议统一数据库变更入口：长期以 `backend/src/main/resources/db/migration` 为唯一事实来源，避免与手工 SQL 演进冲突。
- 建议为 `target/` 等构建产物持续保持忽略，减少无效变更。
- 建议持续补充“目录级文档 + 模块设计文档”，当前项目已具备良好基础（AI、事件驱动、绿色导向文档）。

## 8. 快速定位指南

- 想改接口：看 `backend/src/main/java/org/com/repair/controller`。
- 想改业务规则：看 `backend/src/main/java/org/com/repair/service`。
- 想改表结构演进：看 `backend/src/main/resources/db/migration`。
- 想改页面：看 `frontend/src/views`。
- 想改复用组件：看 `frontend/src/components`。
- 想补数据库初始化与触发器：看 `SQL/Init` 与 `SQL/Trigger`。

以上结构说明可以作为项目交接文档、二开导览和新人入门指南使用。
