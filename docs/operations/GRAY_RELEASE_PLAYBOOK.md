# 灰度发布执行模板（Backend）

适用范围：backend 服务
最后更新：2026-03-31

## 1. 发布前准入条件（必须全部满足）

1. `release-gate` 通过：

```powershell
Set-Location .\backend
.\scripts\release-gate.ps1 -BaseUrl http://localhost:8080
```

2. 回滚版本与回滚人已明确。
3. 迁移脚本在预发演练通过（含回滚演练）。
4. 值班工程师与业务接口人已在发布群就位。

## 2. 灰度阶段与放量策略

### 阶段 A：10% 流量（观察 15 分钟）

放量后重点观察：

1. 5xx 比例（5分钟）
2. P95 延迟（10分钟）
3. 数据库连接池使用率
4. 关键事件码异常数量（如 `MAINTENANCE_ALERT_SCAN_FAILURE`）

进入下一阶段条件（全部满足）：

1. `5xx_rate_5m < 1%`
2. `api_p95_ms_10m < 1200`
3. 未出现 P1 告警

### 阶段 B：30% 流量（观察 20 分钟）

继续观察相同指标，并追加：

1. 关键业务接口成功率（登录、工单、车辆、库存）
2. 错误日志 TopN 是否出现新错误簇

进入下一阶段条件（全部满足）：

1. `5xx_rate_5m < 1.5%`
2. `api_p95_ms_10m < 1500`
3. 数据一致性抽样无异常

### 阶段 C：100% 全量（观察 30-60 分钟）

全量后继续监控并执行发布后自动冒烟：

```powershell
Set-Location .\backend
.\scripts\post-release-smoke.ps1 -BaseUrl http://localhost:8080
```

## 3. 回滚触发判据

任一满足即回滚：

1. `5xx_rate_5m >= 2%` 且持续 10 分钟。
2. 核心接口连续冒烟失败 2 次。
3. 出现数据一致性错误（订单状态错乱、奖励重复入账等）。
4. P1 告警未在 10 分钟内止损。

## 4. 回滚步骤（标准化）

1. 立即停止继续放量。
2. 负载切回上一稳定版本。
3. 确认数据库迁移是否需要执行回滚脚本。
4. 执行自动冒烟确认恢复。
5. 发布群同步：时间点、影响范围、回滚结果。

## 6. 发布后复盘最小清单（24h内）

1. 告警总量与分级分布。
2. 核心接口延迟与错误率对比（发布前后）。
3. 用户反馈与客服工单异常聚类。
4. 后续修复项进入架构债台账。
